เอกสารนี้บันทึกข้อบกพร่องทั้งหมดที่พบและแก้ไขระหว่างการพัฒนาระบบจัดการห้องสมุด v2 โปรเจกต์มีปัญหา 10 ประเด็นหลักที่เกี่ยวข้องกับคุณสมบัติที่หายไป ความไม่สอดคล้องของการแมปฟิลด์ข้อผิดพลาดในการกำหนดค่าเส้นทาง และการเสียหายของไฟล์

### 🔴 BUG #1: หน้าเพจนำทางหายไป

**สถานะ:** ✅ แก้ไขแล้ว  
**ความรุนแรง:** สูง  
**หมวดหมู่:** คุณสมบัติที่หายไป

#### คำอธิบาย

หน้าแดชบอร์ดแสดงปุ่มนำทาง (หนังสือ สมาชิก การยืม รายงาน) แต่หน้า HTML ที่สอดคล้องกันไม่มีอยู่ การคลิกปุ่มเหล่านี้ส่งผลให้เกิดข้อผิดพลาด 404

#### สาเหตุหลัก

ไฟล์มุมมอง HTML ไม่ได้สร้างขึ้นแม้ว่าเส้นทาง API มีอยู่

#### ผลกระทบ

- ผู้ใช้ไม่สามารถเข้าถึงคุณสมบัติหลักของแอปพลิเคชัน
- แอปพลิเคชันไม่สมบูรณ์
- การนำทางเสีย ทันทีหลังจากเข้าสู่ระบบ

#### วิธีแก้ไข

สร้างไฟล์ HTML ที่หายไป:

- `views/books.html` - หน้าจัดการหนังสือ
- `views/members.html` - หน้าจัดการสมาชิก
- `views/borrowing.html` - หน้าจัดการการยืม
- `views/reports.html` - หน้ารายงาน

เพิ่มเส้นทาง GET ใน `src/app.js`:

```javascript
app.get("/dashboard", (req, res) => res.render("dashboard"));
app.get("/books", (req, res) => res.render("books"));
app.get("/members", (req, res) => res.render("members"));
app.get("/borrowing", (req, res) => res.render("borrowing"));
app.get("/reports", (req, res) => res.render("reports"));
```

### 🔴 BUG #2: ความไม่ตรงกันของฟิลด์ ID หนังสือ (การใช้นามแฝง Primary Key)

**สถานะ:** ✅ แก้ไขแล้ว  
**ความรุนแรง:** สูง  
**หมวดหมู่:** การแมปฟิลด์

#### คำอธิบาย

Books API ส่งคืน `book_id` เป็นคีย์หลัก แต่โค้ด JavaScript ฝั่งไคลเอนต์คาดหวัง `id` เมื่อผู้ใช้พยายามแก้ไขหนังสือ ฟังก์ชันแก้ไขไม่สามารถหาหนังสือได้เพราะมองหา `book.id` แต่ API ส่งคืน `book.book_id`

#### ข้อความข้อผิดพลาด

```
400 Bad Request: Book not found
หรือ
Edit modal ไม่เติมข้อมูลหนังสือ
```

#### สาเหตุหลัก

ฐานข้อมูลใช้ `book_id` เป็นชื่อคอลัมน์ แต่มาตรฐาน REST API คาดว่าจะใช้ `id` สำหรับคีย์หลัก โค้ดไคลเอนต์เขียนตามมาตรฐาน REST

#### ผลกระทบ

- **ฟังก์ชันแก้ไขหนังสือ:** ❌ เสียหายอย่างสิ้นเชิง
- ไม่สามารถแก้ไขหนังสือได้เพราะ API หาไม่เจอ
- ผู้ใช้ได้รับข้อผิดพลาด "ไม่พบ" เมื่อเลือกตัวเลือกแก้ไข
- ตัวเลือกไประบายหนังสือในการยืมล้มเหลว

#### วิธีแก้ไข

อัปเดต `src/models/Book.js` เพื่อใช้นามแฝง `book_id` เป็น `id` ในการสืบค้น SELECT ทั้งหมด:

```javascript
// ก่อน
SELECT * FROM books

// หลัง
SELECT book_id as id, * FROM books
```

นำไปใช้กับวิธี:

- `getAll()` - แสดงรายการหนังสือทั้งหมด
- `findById(bookId)` - รับรายละเอียดหนังสือเดียว
- `findByIsbn(isbn)` - ค้นหาตาม ISBN
- `search(query)` - ค้นหาหนังสือ
- `getAvailableBooks()` - รับหนังสือที่สามารถยืมได้

#### การปรับปรุงไคลเอนต์

รายการหนังสือในไระบายทำงานได้ถูกต้องตอนนี้:

```javascript
loadBooks() {
  books.forEach(book => {
    // ใช้ book.id (นามแฝงจาก book_id)
    option.value = book.id;
  })
}
```

### 🔴 BUG #3: ความไม่ตรงกันของฟิลด์ ID สมาชิก

**สถานะ:** ✅ แก้ไขแล้ว

### 🔴 BUG #4: ความไม่ตรงกันของการแมปฟิลด์การยืม

**สถานะ:** ✅ แก้ไขแล้ว  
**ความรุนแรง:** สูง  
**หมวดหมู่:** การแมปฟิลด์ / รูปแบบการตอบสนอง API

#### คำอธิบาย

Borrowing API มีความไม่ตรงกันของชื่อฟิลด์หลายรายการ:

- คอลัมน์ฐานข้อมูล `borrow_id` แต่ API คาดหวัง `id`
- คอลัมน์ฐานข้อมูล `member.full_name` แต่ API ส่งคืนเป็น `full_name` (ไม่ได้ใช้นามแฝง)
- คอลัมน์ฐานข้อมูล `books.title` แต่ API ส่งคืนเป็น `title` (ไม่ได้ใช้นามแฝง)

ตารางการยืมแสดงค่า undefined และคุณสมบัติ View Details ไม่ทำงาน

#### อาการข้อผิดพลาด

- ตารางบันทึกการยืมแสดง "undefined" ในคอลัมน์สมาชิกและหนังสือ
- ปุ่ม View Details ไม่ทำงาน - "ไม่พบบันทึกการยืม"
- ปุ่มคืนหนังสือให้ข้อผิดพลาดเพราะไม่สามารถระบุบันทึกได้

#### สาเหตุหลัก

การสืบค้น `Borrowing.js` ไม่มี JOIN ที่เหมาะสมกับชื่อสมาชิกและหนังสือ และไม่ได้ใช้นามแฝงคีย์หลัก

#### ผลกระทบ

- ❌ **แสดงรายการการยืม** - ชื่อสมาชิกและหนังสือแสดงเป็น undefined
- ❌ **View Details** - เإนพอยต์ GET /borrowing/:borrowId ไม่มีอยู่
- ❌ **คืนหนังสือ** - ไม่สามารถระบุบันทึกที่จะอัปเดต
- ❌ การแสดงข้อมูลเสียหายสิ้นเชิง

#### วิธีแก้ไข

อัปเดตวิธี findById() ใน `src/models/Borrowing.js` เพื่อรวม JOIN และนามแฝง:

```javascript
// ก่อน
SELECT * FROM borrowing WHERE borrow_id = ?

// หลัง
SELECT
  b.borrow_id as id,
  b.*,
  m.full_name as member_name,
  bk.title as book_title
FROM borrowing b
JOIN members m ON b.member_id = m.member_id
JOIN books bk ON b.book_id = bk.book_id
WHERE b.borrow_id = ?
```

อัปเดต `src/controllers/BorrowingController.js` เพื่อเพิ่มวิธี `findById()` ที่หายไป

### 🔴 BUG #6: ความไม่ตรงกันของเมธอด HTTP คืนหนังสือ

**สถานะ:** ✅ แก้ไขแล้ว  
**ความรุนแรง:** สูง  
**หมวดหมู่:** เมธอด HTTP / การกำหนดค่าเส้นทาง

#### คำอธิบาย

เอนดพอยต์คืนหนังสือถูกกำหนดเป็น `POST /borrowing/:borrowId/return` แต่ไคลเอนต์ส่งคำขอ `PUT` สิ่งนี้ทำให้เกิดข้อผิดพลาด "404 Not Found" เมื่อผู้ใช้คลิกปุ่มคืน

#### ข้อความข้อผิดพลาด

```
PUT /borrowing/1/return - 404 Not Found
```

#### สาเหตุหลัก

เส้นทางถูกกำหนดด้วย POST เมื่อจุดประสงค์คือการอัปเดตทรัพยากร (ซึ่งควรใช้ PUT)

#### ผลกระทบ

- ❌ **ฟังก์ชันคืนหนังสือ:** เสียหายสิ้นเชิง
- ผู้ใช้ไม่สามารถคืนหนังสือที่ยืมมา
- ปุ่มคืนแสดงข้อผิดพลาดแทนการประมวลผล
- ไม่มีวิธีใดที่จะทำเครื่องหมายหนังสือเป็นคืนแล้ว

#### วิธีแก้ไข

เปลี่ยนเมธอด HTTP ใน `src/routes/borrowing.js`:

```javascript
// ก่อน
router.post("/:borrowId/return", BorrowingController.returnBook);

// หลัง
router.put("/:borrowId/return", BorrowingController.returnBook);
```

PUT เป็นเมธอด HTTP ที่ถูกต้องสำหรับการอัปเดตสถานะการคืนทรัพยากร

#### โค้ดไคลเอนต์ (ถูกต้องแล้ว)

```javascript
async function returnBook(id) {
  const response = await fetch(`/api/borrowing/${id}/return`, {
    method: "PUT", // เมธอดที่ถูกต้อง
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify({
      returnDate: new Date().toISOString().split("T")[0],
    }),
  });
}
```

### 🔴 BUG #7: เอนดพอยต์ Borrowing Detail หายไป

**ความรุนแรง:** สูง

#### คำอธิบาย

ไคลเอนต์มีปุ่ม "View Details" ในบันทึกการยืมที่เรียก `GET /borrowing/:borrowId` แต่เอนดพอยต์นี้ไม่มีในแบ็กเอนด์ ผู้ใช้ไม่สามารถดูข้อมูลโดยละเอียดเกี่ยวกับบันทึกการยืมเฉพาะものได้

#### ข้อความข้อผิดพลาด

```
GET /borrowing/1 - 404 Not Found
```

#### สาเหตุหลัก

เอนดพอยต์ไม่เคยถูกใช้งานแม้ว่าไคลเอนต์มีโค้ดเพื่อใช้มันอยู่แล้ว

#### ผลกระทบ

- ❌ **ปุ่ม View Details:** ไม่ทำงานเลย
- ผู้ใช้ไม่สามารถเห็นข้อมูลการยืมโดยละเอียด
- ไม่มีวิธีตรวจสอบวันที่ยืม/คืน และจำนวนเงินค่าปรับ

#### วิธีแก้ไข

เพิ่มเอนดพอยต์ที่หายไปใน `src/routes/borrowing.js`:

```javascript
// เส้นทางใหม่
router.get("/:borrowId", BorrowingController.findById);
```

ติดตั้งวิธี `findById()` ใน `src/controllers/BorrowingController.js`:

```javascript
async findById(req, res) {
  try {
    const { borrowId } = req.params;
    const borrowing = await Borrowing.findById(borrowId);

    if (!borrowing) {
      return res.status(404).json({ error: 'ไม่พบบันทึกการยืม' });
    }

    res.json(borrowing);
  } catch (error) {
    res.status(500).json({ error: error.message });
  }
}
```

### 🔴 BUG #8: ปัญหาลำดับเส้นทาง (Express Router Precedence)

**สถานะ:** ✅ แก้ไขแล้ว  
**ความรุนแรง:** สูง  
**หมวดหมู่:** ตรรมชาติของแอปพลิเคชัน / การกำหนดค่าเส้นทาง

#### คำอธิบาย

ใน Express.js เส้นทางจะจับคู่ตามลำดับ เส้นทางทั่วไปเช่น `/:borrowId` ถูกกำหนดก่อนเส้นทางที่เจาะจง เช่น `/borrowed` หรือ `/member/:memberId` ทำให้เส้นทางทั่วไปจับคู่ก่อน ตัวอย่างเช่น เมื่อดึงข้อมูล `/borrowing/borrowed` Express จับคู่เป็น `/:borrowId` ที่มี `borrowId = "borrowed"` ส่งคืนข้อผิดพลาดแทนหนังสือที่ยืมมา

#### อาการข้อผิดพลาด

- การคลิก "View Borrowed" หรือตัวเลือกตัวกรองให้ข้อผิดพลาด
- ประวัติการยืมของสมาชิกไม่แสดง
- การดึงข้อมูลสำหรับมุมมองที่ได้รับการกรองเฉพาะล้มเหลว

#### สาเหตุหลัก

คำจำกัดความของเส้นทางใน `src/routes/borrowing.js` ผิดลำดับ:

```javascript
// ลำดับที่ผิด
router.get('/:borrowId', ...);        // ทั่วไป - จับคู่อะไรก็ได้
router.get('/borrowed', ...);         // เจาะจง - ไม่เคยถึง!
router.get('/member/:memberId', ...); // เจาะจง - ไม่เคยถึง!
```

#### ผลกระทบ

- ❌ มุมมองหนังสือที่ยืมมา: เสียหาย
- ❌ มุมมองหนังสือค้างชำระ: เสียหาย
- ❌ ประวัติการยืมของสมาชิก: เสียหาย
- เฉพาะการดำเนินการรายการทั่วไปและเพิ่มเท่านั้นที่ทำงาน

#### วิธีแก้ไข

เรียงลำดับเส้นทางใหม่ใน `src/routes/borrowing.js` เพื่อใส่เส้นทางเจาะจงบนเส้นทางทั่วไป:

```javascript
// ลำดับที่ถูกต้อง
router.get('/borrowed', ...);         // เจาะจงก่อน
router.get('/overdue', ...);          // เจาะจงก่อน
router.get('/member/:memberId', ...); // เจาะจงก่อน
router.get('/', ...);                 // รายการทั่วไปล่าสุด
router.post('/', ...);                // สร้างทั่วไป
router.get('/:borrowId', ...);        // รายละเอียดทั่วไป
router.put('/:borrowId/return', ...); // อัปเดตทั่วไป
```

สิ่งนี้แน่ใจว่า Express ประเมินเส้นทางเจาะจงก่อนพยายามจับคู่รูปแบบ `/:borrowId` ทั่วไป

### 🔴 BUG #9: ความเสียหายของไฟล์ Members.html

**สถานะ:** ✅ แก้ไขแล้ว  
**ความรุนแรง:** วิกฤต  
**หมวดหมู่:** ความเสียหายของไฟล์

#### คำอธิบาย

ไฟล์ `views/members.html` เสียหายเนื่องจากมีโค้ด HTML และ JavaScript สลับผสมกัน แท็กปิดแบบฟอร์มปรากฏอยู่ในตัวฟังก์ชัน JavaScript และไวยากรณ์ JavaScript ผสมกับองค์ประกอบ HTML ไฟล์เสียหายสิ้นเชิงและใช้งานไม่ได้

#### อาการ

- เบราว์เซอร์ console แสดงข้อผิดพลาด JavaScript syntax
- หน้าสมาชิกไม่โหลดหรือแสดงผล blank
- ฟีเจอร์การจัดการสมาชิกทั้งหมด (รายการ เพิ่ม แก้ไข ลบ) ไม่ทำงาน

#### สาเหตุหลัก

การดำเนินการ `replace_string_in_file` ตามลำดับไม่สามารถรักษาขอบเขตโค้ดที่เหมาะสม โค้ด HTML เก่าและโค้ด JavaScript ใหม่ผสมกันระหว่างการแทนที่ทำให้เสียหายโครงสร้าง

#### ผลกระทบ

- ❌ **หน้าสมาชิก:** ไม่ทำงานเลย
- ❌ ไม่มีวิธีดูรายชื่อสมาชิก
- ❌ ไม่มีวิธีเพิ่ม/แก้ไข/ลบสมาชิก
- ❌ ฟีเจอร์การยืมสมาชิกไม่สามารถทำงานได้หากไม่มีข้อมูลสมาชิก

#### วิธีแก้ไข

เขียนใหม่อย่างสมบูรณ์ `views/members.html` ด้วย:

1. โครงสร้าง HTML ที่สะอาดพร้อมเลย์เอาต์ Bootstrap ที่เหมาะสม
2. ฟังก์ชัน JavaScript ที่แยกจากกันอย่างถูกต้อง
3. แบบฟอร์มมอดอล 2 แบบ (เพิ่มสมาชิกและแก้ไขสมาชิก) พร้อมการผูกค่าฟิลด์แบบฟอร์มที่ถูกต้อง
4. ตัวฟัง event สำหรับการดำเนินการ CRUD ทั้งหมด
5. ตารางพร้อมส่วนหัวคอลัมน์ที่เหมาะสมและการผูกข้อมูล

ไฟล์ตอนนี้รวม:

- แถบนำทางพร้อมลิงก์ไปยังทุกหน้า
- ตารางสมาชิกแสดงสมาชิกทั้งหมดพร้อมการดำเนินการ
- มอดอลเพิ่มสมาชิก (แบบฟอร์มบันทึกใหม่)
- มอดอลแก้ไขสมาชิก (แบบฟอร์มอัปเดต)
- การยืนยันลบพร้อมการเรียก API ที่เหมาะสม
- การแสดงสถานะด้วยสัญลักษณ์แสดงสีที่เข้าชุด

### 🔴 BUG #10: ข้อผิดพลาด Jest Configuration Syntax

**สถานะ:** ✅ แก้ไขแล้ว  
**ความรุนแรง:** ปานกลาง  
**หมวดหมู่:** การกำหนดค่า Build/Test

#### คำอธิบาย

ไฟล์ `jest.config.js` ใช้ไวยากรณ์ JSON แทนไวยากรณ์ Node.js module.exports Jest ไม่สามารถโหลดการกำหนดค่าได้อย่างถูกต้อง ทำให้การดำเนินการ test ล้มเหลว หรือใช้การตั้งค่าเริ่มต้น

#### ข้อผิดพลาด

```
SyntaxError: Unexpected token } in jest config
```

#### สาเหตุหลัก

ไฟล์เขียนเป็นวัตถุ JSON แทนโมดูล JavaScript ที่ส่งออกการกำหนดค่า

#### ผลกระทบ

- ❌ ไม่สามารถทำการ Jest test
- ❌ เฟรมเวิร์ก test ไม่มีการกำหนดค่าที่เหมาะสม
- ❌ ไม่สามารถสร้างรายงาน coverage
- ความเชื่อมั่นของโปรเจกต์ไม่สามารถตรวจสอบได้

#### วิธีแก้ไข

เปลี่ยน `jest.config.js` จาก JSON เป็นโมดูล JavaScript ที่เหมาะสม:

```javascript
// ก่อน
{
  "testEnvironment": "node",
  "testMatch": ["**/__tests__/**/*.test.js"],
  "collectCoverageFrom": ["src/**/*.js"],
  "coveragePathIgnorePatterns": ["/node_modules/"]
}

// หลัง
module.exports = {
  testEnvironment: 'node',
  testMatch: ['**/__tests__/**/*.test.js'],
  collectCoverageFrom: ['src/**/*.js'],
  coveragePathIgnorePatterns: ['/node_modules/']
};
```

## บทเรียนที่ได้เรียนรู้

1. **การใช้ Alias ฟิลด์:** เมื่อชื่อคอลัมน์ฐานข้อมูลไม่ตรงกับมาตรฐาน REST API (id) ให้ใช้ SQL alias ในการสืบค้นเสมอ
2. **ลำดับเส้นทาง:** ใน Express.js กำหนดเส้นทางเจาะจงก่อนเส้นทางแบบพารามิเตอร์ทั่วไป
3. **เมธอด HTTP:** ใช้กริยา HTTP ที่ถูกต้อง - POST สำหรับสร้าง PUT สำหรับอัปเดต GET สำหรับดึง DELETE สำหรับลบ
4. **ความสมบูรณ์ของไฟล์:** การแทนที่สตริงตามลำดับหลายรายการสามารถทำให้ไฟล์เสียหายได้ - ใช้การดำเนินการแบบกลุ่ม
5. **เอกสาร API:** บันทึกเอนดพอยต์ทั้งหมดด้วยชื่อฟิลด์ที่ถูกต้องและเมธอด HTTP ตั้งแต่ต้น
6. **รูปแบบการกำหนดค่า:** ตรวจสอบให้แน่ใจว่าไฟล์การกำหนดค่าใช้ไวยากรณ์ที่เหมาะสมสำหรับตัวแยกวิเคราะห์ (Node.js ต้อง module.exports ไม่ใช่ JSON)
